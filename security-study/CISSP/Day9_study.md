## 📖 解説パート

### 対称暗号と非対称暗号

対称暗号は、暗号化と復号に同じ鍵を使用する方式です。処理速度が速いという利点がありますが、鍵の共有方法が課題となります。代表的なアルゴリズムには、AESやDESなどがあります。

一方、非対称暗号は、暗号化と復号に異なる鍵（公開鍵と秘密鍵）を使用します。鍵の共有問題を解決できますが、対称暗号に比べて処理速度が遅いという欠点があります。代表的なアルゴリズムには、RSAや楕円曲線暗号（ECC）などがあります。

### ハッシュ関数

ハッシュ関数は、任意の長さのデータを固定長のハッシュ値に変換する関数です。元のデータからハッシュ値を生成することは容易ですが、ハッシュ値から元のデータを推測することは非常に困難です。ハッシュ関数は、データの完全性検証やパスワードの保管などに利用されます。代表的なアルゴリズムには、SHA-256やSHA-3などがあります。

[コード例：PythonにおけるSHA-256ハッシュ値の生成]
```python
import hashlib

data = "Hello, world!"
hash_object = hashlib.sha256(data.encode())
hex_dig = hash_object.hexdigest()
print(hex_dig)
```

#### 重要ポイント

* 対称暗号は速度が速く、大量のデータ暗号化に適している。
* 非対称暗号は鍵の共有が容易で、認証やデジタル署名に適している。
* ハッシュ関数はデータの完全性検証に利用され、不可逆な特性を持つ。

### 公開鍵基盤（PKI）

公開鍵基盤（PKI）は、公開鍵暗号方式を利用して、デジタル証明書を発行・管理するための仕組みです。PKIを利用することで、通信相手の身元を確実に認証し、安全な通信を行うことができます。PKIの主要な構成要素には、認証局（CA）、登録局（RA）、証明書リポジトリなどがあります。

## 🏢 ケーススタディ

### ケース：ECサイトにおけるクレジットカード情報の保護

ECサイトでは、顧客のクレジットカード情報を安全に保護する必要があります。

#### 問題点

* クレジットカード情報が漏洩した場合、顧客に経済的損害を与える可能性がある。
* 法規制（PCI DSSなど）への違反となる可能性がある。
* 企業ブランドの信頼を失墜させる可能性がある。

#### 対応策

* クレジットカード情報を暗号化して保存する。
* SSL/TLSを使用して、通信経路を暗号化する。
* アクセス制御を厳格に行い、不正アクセスを防止する。
* 定期的な脆弱性診断を実施する。

#### ケースから学ぶ教訓

* 顧客情報を保護することは、企業の信頼維持に不可欠である。
* 暗号化技術は、情報漏洩のリスクを軽減するための重要な手段である。
* 法規制を遵守し、セキュリティ対策を継続的に改善する必要がある。

## ✅ テスト回答・解説

### 問題1 正解：2

解説：対称暗号は暗号化と復号に同じ鍵を使用するため、処理速度が速いという利点があります。鍵の共有方法が課題となります。

### 問題2 正解：3

解説：ハッシュ関数は不可逆な関数であり、暗号化には使用されません。データの完全性検証やパスワードの保管などに利用されます。

### 問題3 正解：4

解説：ファイアウォールはネットワークセキュリティのための要素であり、PKIの構成要素ではありません。

### 問題4 正解：3

解説：RSAは代表的な非対称暗号方式のアルゴリズムです。AESとDESは対称暗号、MD5はハッシュ関数です。

### 問題5 正解：3

解説：クレジットカード情報をすべての従業員に開示することは、情報漏洩のリスクを高めるため、不適切な対策です。
