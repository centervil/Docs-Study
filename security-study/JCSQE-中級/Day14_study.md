## 📖 解説パート

### SQCツールの概要と分類

SQC（Statistical Quality Control：統計的品質管理）は、統計的手法を用いて品質を管理する手法の総称です。データに基づいた客観的な判断を可能にし、品質改善活動を効率的に進めることができます。

SQCツールは、データの種類や分析の目的に応じて様々なものが存在します。代表的なものとしては、管理図、ヒストグラム、パレート図、特性要因図、散布図などが挙げられます。これらのツールは、単独で使用するだけでなく、組み合わせて使用することで、より深い分析が可能になります。

### 主要なSQCツールの詳細

*   **管理図**: 工程が安定しているかどうかを時系列で監視するために使用します。X̄-R管理図、p管理図、c管理図など、データの種類に応じて適切な管理図を選択します。

*   **ヒストグラム**: データの分布を視覚的に把握するために使用します。データの中心位置やばらつき、異常値の有無などを確認することができます。

*   **パレート図**: 問題の重要度を視覚的に把握するために使用します。問題発生件数や損失額などを大きい順に並べることで、重点的に取り組むべき課題を明確にすることができます。

*   **特性要因図**: 問題の原因を特定するために使用します。魚の骨のような図で、問題（特性）に対する要因を洗い出し、整理します。

*   **散布図**: 2つの変数の関係を視覚的に把握するために使用します。相関関係の有無や強さを確認することができます。

#### 重要ポイント

*   SQCツールは、データの種類や分析の目的に応じて適切に選択する必要がある。
*   SQCツールは、単独で使用するだけでなく、組み合わせて使用することで、より深い分析が可能になる。
*   SQCツールを使用する際は、データの収集方法や分析結果の解釈に注意する必要がある。

### 管理図の具体的な活用例

管理図は、工程が安定しているかどうかを監視するために非常に有効なツールです。例えば、製造ラインにおける製品の寸法を定期的に測定し、その結果をX̄-R管理図にプロットすることで、工程の異常を早期に発見することができます。

```
# 例：X̄-R管理図の作成 (Python)
import matplotlib.pyplot as plt
import numpy as np

# データ
data = np.random.normal(10, 1, 100) # 平均10、標準偏差1の正規分布に従うデータ

# 管理限界の計算
mean = np.mean(data)
std = np.std(data)
UCL = mean + 3 * std
LCL = mean - 3 * std

# プロット
plt.plot(data, label='Data')
plt.axhline(UCL, color='red', linestyle='--', label='UCL')
plt.axhline(LCL, color='red', linestyle='--', label='LCL')
plt.axhline(mean, color='green', linestyle='-', label='Mean')
plt.legend()
plt.xlabel('Sample Number')
plt.ylabel('Value')
plt.title('X̄-R Control Chart')
plt.show()
```

## 🏢 ケーススタディ

### ケース：製造ラインにおける不良率の増加

ある製造ラインにおいて、不良率が目標値を上回り、顧客からのクレームが増加しているという問題が発生しました。原因を特定するために、SQCツールを活用することになりました。

#### 問題点

*   不良率が目標値を上回っている。
*   顧客からのクレームが増加している。
*   不良原因が特定できていない。

#### 対応策

*   ヒストグラムを用いて、不良データの分布を把握する。
*   パレート図を用いて、不良の種類とその発生頻度を分析する。
*   特性要因図を用いて、不良の原因を洗い出す。
*   管理図を用いて、工程の安定性を監視する。
*   散布図を用いて、不良と関連性の高い要因を特定する。

#### ケースから学ぶ教訓

*   SQCツールは、問題解決の初期段階で、現状を客観的に把握するために有効である。
*   複数のSQCツールを組み合わせることで、より深い分析が可能になる。
*   データに基づいた分析を行うことで、効果的な対策を立案することができる。

## ✅ テスト回答・解説

### 問題1 正解：3

解説：管理図は、工程が時間とともに安定しているかどうかを監視するために設計されています。管理限界線を超えた点が現れると、工程に異常が発生している可能性を示唆します。ヒストグラムはデータの分布を、パレート図は問題の優先順位を、散布図は二つの変数間の関係性を示します。

### 問題2 正解：2

解説：パレート図は、問題点や原因を発生頻度やコストなどの重要度順に並べた棒グラフです。これにより、最も影響の大きい問題に焦点を当てることができます。

### 問題3 正解：3

解説：特性要因図（魚の骨図）は、特定の問題（特性）に対する潜在的な原因を体系的に洗い出すために使用されます。

### 問題4 正解：4

解説：散布図は、二つの変数間の関係性を視覚的に表現するために使用されます。点の分布パターンから、正の相関、負の相関、または相関がないかどうかを判断できます。

### 問題5 正解：1

解説：ヒストグラムは、データの分布を棒グラフで表現し、データの中心位置、ばらつき、形状などを把握するのに役立ちます。
