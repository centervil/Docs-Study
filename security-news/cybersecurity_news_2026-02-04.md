# 2026年2月4日 サイバーセキュリティニュースまとめ

## 主要ポイント
* **IPA「情報セキュリティ10大脅威 2026」でAIリスクが初選出:** 「AIの利用をめぐるサイバーリスク」が組織部門の3位にランクインし、AIの悪用や不適切利用による脅威が公式に定義されました。
* **国家背景の攻撃者（APT28）によるOffice脆弱性の悪用:** Microsoft Officeの脆弱性（CVE-2026-21509）を突いた諜報活動が、ウクライナや東欧諸国を標的に展開されています。
* **開発ツールおよびAIアシスタントの深刻な脆弱性:** DockerのAIアシスタント「Ask Gordon」におけるコード実行の欠陥や、React Native CLIの「Metro4Shell」脆弱性が相次いで報告されています。

## AIと次世代の脅威
AI技術の統合が進む中、その信頼性を逆手に取った攻撃が顕在化しています。DockerのAIアシスタントに存在する脆弱性（DockerDash）は、AIエージェントに与えられた「文脈上の信頼」を悪用する新たな攻撃のひな型となっています。また、IPAのランキングでAIリスクが上位に入ったことは、AIに対する理解不足による意図せぬ情報漏洩や、攻撃の敷居低下が重大な懸念事項となったことを示しています。

## 国家主導の諜報活動と高度な回避技術
ロシア背景のAPT28は、高度にローカライズされたソーシャルエンジニアリングを用い、MiniDoor（メール窃取）やPixyNetLoaderといったカスタムマルウェアを配布しています。彼らはサーバー側での回避技術やステガノグラフィーを駆使し、標的地域以外のIPアドレスには悪意あるペイロードを送信しないなどの徹底した運用セキュリティを保持しています。

## 国内インシデントとランサムウェア被害
日本国内では、穴吹興産（2月3日発生）や青山メイン企画、エネサンスホールディングスなどがランサムウェア攻撃を受けており、サーバーの暗号化や個人情報の流出が確認されています。また、ソフトバンクではプロキシサーバーの不具合により、設定ミスに起因するプライバシー侵害が発生しました。

### 1. APT28によるMicrosoft Office脆弱性（CVE-2026-21509）の悪用
ロシアに関連するAPT28が、Officeの脆弱性（CVE-2026-21509）を悪用してウクライナ、ルーマニア、スロバキアなどの政府・軍事関係者を標的にしています。攻撃は、軍事訓練プログラムなどを装った悪意あるRTFファイルから始まり、Outlookメールを窃取する「MiniDoor」や、COMオブジェクトハイジャッキングで持続性を確立する「PixyNetLoader」をドロップします。

### 2. Docker、AIアシスタント「Ask Gordon」の深刻な脆弱性を修正
Dockerは、AIアシスタント「Ask Gordon」に存在するコード実行およびデータ窃取が可能な脆弱性を修正しました。この脆弱性（DockerDash）は、Dockerイメージ内の悪意あるLABEL命令をAIが読み取り、検証なしでMCPゲートウェイへ転送してしまうことで、攻撃者がユーザーのDocker特権で任意のコマンドを実行できる仕組みとなっていました。
[出典: Docker Fixes Critical Ask Gordon AI Flaw Allowing Code Execution via Image Metadata]

### 3. IPA「情報セキュリティ10大脅威 2026」：AIリスクが3位に浮上
情報処理推進機構（IPA）は、2025年の動向を反映した最新の脅威ランキングを発表しました。組織部門では「ランサムウェア被害」が1位、「サプライチェーン攻撃」が2位を維持する中、初選出の「AIの利用をめぐるサイバーリスク」が3位にランクインしました。これには、AIの不適切利用による権利侵害や、AI悪用による攻撃の巧妙化などが含まれています。
[出典: 「セキュリティ10大脅威2026」発表 - 多岐にわたる脅威「AIリスク」が初選出 - Security NEXT]

## 深掘り
### AIエージェントへの「文脈上の信頼」の崩壊
近年の開発パイプラインやオペレーティングシステムへのLLM（大規模言語モデル）の深い統合により、従来のコードの脆弱性とは異なる「文脈」を突く攻撃が登場しています。

#### DockerDashに見るインジェクション攻撃
DockerDashのインシデントは、一見無害なメタデータ（LABELフィールド）がAIによって処理される際に、悪意ある命令として解釈される危険性を示しました。AIは、それが単なる説明文なのか、実行すべき命令なのかを区別する能力が不十分であり、中間層であるMCPゲートウェイがAIからのリクエストを「信頼されたソース」として盲目的に受け入れてしまったことが、コード実行を許す決定的な要因となりました。

#### AIサプライチェーンの汚染
APT28やLotus Blossom（Billbug）のような国家背景の攻撃者は、AIアシスタントや開発ツールなどの「信頼されたインフラ」を武器化するスピードを上げています。例えば、Notepad++の配布インフラを侵害してカスタムマルウェアChrysalisを配布するなどの事例もあり、ツールそのものへの信頼がサプライチェーン全体の汚染に繋がっています。

### [関連知識]
#### [用語解説]
*   **COMオブジェクトハイジャッキング:** OSの正規のCOMコンポーネントの動作を乗っ取り、マルウェアが自動起動する設定を埋め込む技術。
*   **ステガノグラフィー:** 画像ファイル（SplashScreen.pngなど）の中に秘密裏にシェルコードなどのデータを隠蔽する手法。
*   **Living off the land (LotL):** 攻撃者が独自のツールを持ち込むのではなく、標的システム上の正規ツールやクラウドサービス（filen.ioなど）を悪用して活動する手法。

#### [今後の展望]
IPAがAIリスクをトップ3に掲げたことは、セキュリティ専門家にとって大きな転換点となります。今後は、コードの静的解析だけでなく、AIエージェントに与えられている権限や、AIが外部データ（メタデータ等）を処理する際の「コンテキストの検証」を評価する新たなセキュリティ基準の構築が急務となります。

## 結論
2026年2月の情勢は、サイバー環境がますます「非対称的」になっていることを示しています。国家背景の攻撃者が高度な回避技術と信頼されたツールを武器化する一方で、国内組織はランサムウェアや基本的な設定ミス（プロキシ不具合等）による被害に直面し続けています。AIエージェントが開発現場に浸透する中、利便性と引き換えに生じる新たな攻撃経路を早期に評価し、防御を固めることが不可欠です。
