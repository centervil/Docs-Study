# 2025年12月8日 サイバーセキュリティニュースまとめ
## 主要ポイント
*   **React Server Components (RSC) のRCE脆弱性（CVE-2025-55182）**が公開から**「数時間以内」**に中国国家関連グループによって悪用され、ゼロデイ攻撃の超高速化が深刻化しています。
*   EUサイバーレジリエンス法（CRA）や米国の医療サイバーセキュリティ法案により、脆弱性の悪用確認後**24時間以内の当局報告**や、MFA、暗号化、ペネトレーションテストなどの**最低セキュリティ基準の義務化**が国際的に進んでいます。
*   国内では、三菱UFJ銀行が能動的セキュリティ技術を取り込む合弁会社設立を報じられるなど、高度化する脅威に対応するため、防御から**プロアクティブな「セキュリティ保証」**への戦略転換が加速しています。

## ゼロデイ脆弱性の緊急事態
CVSSスコア10.0のReact2Shell脆弱性(CVE-2025-55182)は、React Server Components (RSC) のFlightプロトコルにおける非安全な逆シリアル化に起因する、認証不要のリモートコード実行（RCE）の欠陥です。Amazonの脅威インテリジェンスチームは、この脆弱性が公開されてから**「数時間以内」**に、Earth LamiaやJackpot Pandaといった中国国家関連の脅威グループによる積極的な悪用試行を観測しました。この極端な悪用までの時間短縮は、従来のパッチ適用戦略では対応が間に合わないことを示しており、CISAは連邦機関に対して2025年12月26日までの対応期限を課しています。

## 国際的なセキュリティ規制の強化
EUでは、EU市場で販売されるデジタル製品の製造者に対し、製品の脆弱性が悪用されたことを確認した場合、**24時間以内**に該当機関に開示することを義務付ける欧州サイバーレジリエンス法案（CRA）が成立に向けて大詰めを迎えています。また、米国では医療・公衆衛生部門（HPH）のセキュリティ強化を目的とした「Health Care Cybersecurity and Resiliency Act of 2025」が再提出され、HIPAA規制を更新し、**多要素認証（MFA）と暗号化の採用**、および**ペネトレーションテストを含む監査の実施**を義務的な最低基準とする方針が示されています。

## 新型マルウェアとサプライチェーンリスク
2025年には、検知を回避する巧妙な新型マルウェアが確認されています。リモートアクセス型マルウェア**「FINALDRAFT」**は、Microsoft Outlookの下書き機能を悪用した秘匿通信を行い、正規トラフィックに偽装することで検知を回避します。また、Desert Dexterグループによる**AsyncRATマルウェアキャンペーン**では、Facebook広告やDropbox、Telegramなどの**正規のWebサービスを悪用**し、ユーザーに安心感を与える形でマルウェアが配布される多段階型の拡散手法が特徴です。さらに、2025年のデータ侵害事例の多くは、サードパーティの脆弱性やソーシャルエンジニアリングに起因しており、サプライチェーンの弱点が主要な脅威となっています。

## ニュース紹介
### 1. React2Shell RCE脆弱性 (CVE-2025-55182)の公開と超高速悪用
React Server Components (RSC) のRCE脆弱性（React2Shell/CVE-2025-55182）は、CVSSスコア10.0の最高危険度を持つ認証不要な脆弱性です。この問題は、ReactのFlightプロトコルにおける**非安全な逆シリアル化**（Insecure Deserialization）という構造的欠陥が根本原因です。この欠陥により、攻撃者は悪意のあるHTTPリクエストを送信するだけでサーバー上で任意のコードを実行できてしまいます。Amazonの脅威インテリジェンスチームは、脆弱性公開から**「数時間以内」**に、Earth LamiaやJackpot Pandaを含む中国国家関連の脅威グループによる積極的な悪用試行を確認しました。影響を受けるのは`react-server`パッケージの各種ライブラリや、Next.jsなどの主要なダウンストリームフレームワークです。CISAは連邦政府機関に対し、2025年12月26日までにパッチ適用を義務付けています。

### 2. 2025年に確認された新型マルウェアの動向
2025年に入り、検知回避が巧妙化し、一般的なセキュリティ対策では防ぎきれない新型マルウェアが増加しています。リモートアクセス型マルウェア**「FINALDRAFT」**は、WindowsとLinuxの両方に対応し、Microsoft 365の**Outlookの下書き機能**を利用して、メール送信を伴わない秘匿的なC2通信（コマンド＆コントロール通信）を実現します。この通信は正規のMicrosoft Graph APIトラフィックに偽装され、検知されにくいのが特徴です。また、サイバー犯罪グループDesert Dexterによる**AsyncRATマルウェアキャンペーン**では、Facebook広告やDropboxといった**正規のWebサービスを悪用**し、利用者に安心感を与えつつ多段階的にマルウェアを拡散させる手法が確認されました。

### 3. 国際的なセキュリティ規制：報告義務の24時間化と最低基準の義務化
EUでは、デジタル製品の製造者に対し、脆弱性の悪用を確認した場合、**24時間以内**に当局へ早期警告通知を義務付ける欧州サイバーレジリエンス法案（CRA）の施行が予定されています。一方、米国では、医療・公衆衛生部門のセキュリティを強化する「Health Care Cybersecurity and Resiliency Act of 2025」が再提出されました。この法案は、HIPAA規制を更新し、**多要素認証（MFA）**、**暗号化**、**ペネトレーションテストを含む監査の実施**といった具体的な技術要件を、法的な最低基準として義務付ける方針を示しています。これらの国際動向は、インシデント対応の速度（24時間以内）と、セキュリティ技術のガバナンスレベル（MFA、暗号化必須）をグローバル標準として引き上げています。

## 深掘り
### ゼロデイ攻撃の超高速化がもたらすセキュリティ戦略の転換
#### サブトピック1
TTE（Time-to-Exploit）の劇的な短縮と防御戦略の限界
React2Shellの事例は、攻撃者が新規脆弱性の公開から運用化までの時間（TTE）を「数時間以内」に劇的に短縮しているという、新たな脅威環境の現実を示しています。Amazonの脅威インテリジェンスチームは、特定の脅威クラスターが悪用試行のトラブルシューティングを体系的に行っていた詳細な記録を観測しており、防御側に残されたパッチ適用の猶予が数時間単位へと劇的に収縮していることを意味します。この状況は、従来の平均パッチ適用期間（MTTP）が脅威の速度に追いついていないことを示しており、企業には、パッチ適用が遅延した場合に備えた侵害後の検出体制（MDR/XDR）の監視レベルを最大限に引き上げることが不可欠です。

#### サブトピック2
サプライチェーンとサードパーティリスクの顕在化
React2Shell脆弱性がNext.jsなどへの依存性を通じてサプライチェーン全体に影響を及ぼすように、サプライチェーンの弱点を悪用した攻撃は、ランサムウェアによる被害と並んで情報セキュリティの主要な脅威となっています。2025年に発生したデータ侵害事例でも、Qantas（第三者コンタクトセンター）、TransUnion（第三者アプリケーション）、Workday（CRMプラットフォーム）など、多くの被害がサードパーティの脆弱性やソーシャルエンジニアリングに起因しています。経済産業省は、利用するソフトウェア部品の影響範囲を迅速に特定できるよう、**ソフトウェア部品構成表（SBOM）の活用**を促進しており、これにより脆弱性の存在を即座に認識し、対応開始までの時間を短縮することが可能です。

### 関連知識
#### 用語解説
*   **React2Shell (CVE-2025-55182):** React Server Components (RSC) に存在する、CVSSスコア10.0の認証不要なリモートコード実行（RCE）の脆弱性の通称です。根本原因はFlightプロトコルにおける**非安全な逆シリアル化**にあります。
*   **非安全な逆シリアル化 (Insecure Deserialization):** 外部からの信頼できないデータ入力が、アプリケーションのメモリ内で実行可能なオブジェクトに変換される過程で、悪意あるコードを実行可能にしてしまう構造的な欠陥です。これは「最も危険なクラスのソフトウェア脆弱性の一つ」と指摘されています。
*   **欧州サイバーレジリエンス法（CRA: Cyber Resilience Act）:** EUで審議されている法律で、「デジタルの要素を持つ製品」の製造者に対し、セキュリティ対策や、脆弱性の悪用確認後24時間以内の当局報告を義務付けるものです。

#### 今後の展望
国際的な規制要件の厳格化と脅威の高速化に対応するため、企業は従来の防御的な姿勢から、**能動的なセキュリティ保証（Proactive Assurance）**モデルへと戦略を転換する必要があります。これは、三菱UFJ銀行が高度な攻撃的セキュリティ技術を取り込んだ合弁会社を設立する動きにも示されています。今後は、EU CRAや米国のHIPAA改正案が要求するMFA、暗号化、ペネトレーションテストなどの**国際的な最低技術基準**を、グローバル事業を展開する上でのコンプライアンス要件として採用することが求められます。また、インシデント発生時の**24時間報告義務**に対応するため、迅速なフォレンジックと影響範囲特定が可能な外部専門サービスとの緊急時連携体制の構築が喫緊の課題です。

## 結論
現在のサイバー脅威環境は、ゼロデイ脆弱性が公開から数時間で悪用される「時間軸の崩壊」に特徴づけられています。企業は、この超高速の脅威と、24時間以内の報告を要求する国際的な規制強化という二重の課題に直面しています。安全なデジタル化を継続するためには、サプライチェーンリスクの管理（SBOMの活用）と、MFAや暗号化といった国際的な最低技術基準の遵守に加えて、**能動的なセキュリティ検証**と**24時間体制の迅速なインシデント対応能力**を確保するための戦略的な投資が不可欠です。
