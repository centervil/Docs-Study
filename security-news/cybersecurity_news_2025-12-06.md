## グローバル脅威/ゼロデイ脆弱性
React Server Components (RSC) の「Flight」プロトコルにおける最高深刻度（CVSS 10.0）のリモートコード実行（RCE）の脆弱性「React2Shell」（CVE-2025-55182）が、本日付で主要な脅威として浮上しています,。この脆弱性は、認証なしで悪用が可能であり、Next.jsなどのダウンストリームフレームワークにも影響を与えます,,。積極的に悪用されている証拠に基づき、CISAはこれを既知の悪用済み脆弱性（KEV）カタログに直ちに追加しました,。Amazonの報告によると、脆弱性公開からわずか数時間以内に、Earth LamiaやJackpot Pandaなどの中国関連APTアクターからの攻撃試行が観測されています。既に30以上の組織が侵害を受け、77,000を超えるインターネットに公開されたIPアドレスが脆弱な状態にあります。
## 国内インシデント/情報漏えい
ホビー商品販売の駿河屋は、2025年8月に公表した不正アクセス事案に関するフォレンジック調査の最終結果を公表しました,,。2025年7月23日から8月6日の期間に通販サイト「駿河屋.jp」でカード決済を行った利用者について、クレジットカード情報30,431件と顧客情報29,932人分の流出可能性が確定しました,。流出した情報には、カード番号、有効期限、名義人名、カードのブランドに加え、**セキュリティコード**が含まれていたことが判明し、大規模な不正利用のリスクとPCI DSSコンプライアンス上の重大な問題を示しています,,。
## ソーシャルエンジニアリング/詐欺
国内では、著名ブランドの信頼性を悪用した詐欺活動が継続的に確認されています。みずほ証券をかたり、「お客様情報の更新に関するお願い」などの件名で機密情報を詐取しようとするフィッシングメールが確認されているほか,、外食チェーンの丸亀製麺の公式Xアカウントをかたるなりすましが発生し、偽のキャンペーン当選連絡を通じて個人情報を窃取しようとする手口も確認されています,。これらの攻撃は、従来のメール防御を回避し、幅広いターゲット層を狙う高度なターゲティング戦略を採用していることを示唆しています。

### 1. Critical React, Next.js flaw lets hackers execute code on servers
最大深刻度の脆弱性「React2Shell」（CVE-2025-55182）がReact Server Components (RSC) の「Flight」プロトコルで発見されました。この脆弱性はリモートコード実行（RCE）を可能にし、認証なしで悪用される可能性があります,。攻撃者は既に活発にこの脆弱性を悪用しており、研究者は、77,000を超えるインターネットに公開されたIPアドレスが脆弱であり、既に30以上の組織が侵害を受けたと確認しています。脆弱性公開後わずか数時間で、中国関連の脅威アクターが攻撃試行を開始しました,。これを受けてCISAは本脆弱性を既知の悪用済み脆弱性（KEV）カタログに追加し、緊急の対応を促しています,。

## 深掘り
### React2Shell RCE脆弱性の緊急性と技術的背景
最高深刻度RCE脆弱性であるReact2Shell（CVE-2025-55182）は、現代のソフトウェアサプライチェーンの脆弱性と、脅威アクターの対応速度の劇的な上昇を象徴しています,。
#### 安全でない逆直列化（Insecure Deserialization）
React2Shellの根本原因は、React Server Componentsのサーバーとクライアント間の通信プロトコルである**Flightプロトコル**における「安全でない逆直列化」にあります,。この脆弱性は、Reactがペイロードをデコードする際の欠陥を利用するものであり、認証されていないリモート攻撃者が任意のコマンド実行を可能にします,。Bitdefenderの技術ソリューションディレクターは、テキストをオブジェクトに変換する処理は、ソフトウェア脆弱性の中でも**「最も危険なクラスの一つ」**であると指摘しています,。
#### 中国系APTグループの即時悪用
この脆弱性の緊急性を高めているのは、その悪用速度です。Amazonは、脆弱性公開から**わずか数時間以内**に、Earth LamiaやJackpot Pandaといった中国関連のAPTグループのインフラからの攻撃試行を観測したと報告しています。この速度は、従来のパッチ管理プロセスでは防御が追いつかないことを明確に示しており、組織は数時間単位での対応を可能とする「緊急セキュリティ運用体制」の構築が求められます,。
### 関連知識
#### 用語解説
*   **RCE (Remote Code Execution / リモートコード実行):** 遠隔の攻撃者がネットワーク経由で標的のシステム上で任意のコードを実行できる脆弱性、またはその能力を指します,。
*   **CISA KEV (Known Exploited Vulnerabilities):** 米国サイバーセキュリティ・インフラストラクチャー安全保障庁が公開する、既に攻撃者が積極的に悪用していることが確認された脆弱性のリストです。このリストへの追加は、緊急の修正対応が必要であることを意味します,。
*   **安全でない逆直列化 (Insecure Deserialization):** 信頼できないソースからのシリアライズされたデータを、プログラム内の実行可能なオブジェクトに変換する際に、入力検証が不十分なために、悪意のあるオブジェクト構造が実行されてしまう脆弱性です,,。
#### 今後の展望
React2Shellのような脅威に対応するため、組織はセキュリティ戦略を根本的に見直す必要があります。脅威インテリジェンスのリアルタイム化（数時間単位の対応）と、アセットインベントリとの自動照合が急務です,。また、駿河屋の事例が示すように、EC事業者は決済情報保護のため、サーバーサイドだけでなく、ウェブサプライチェーン上の第三者スクリプトの脆弱性や改ざんに対処する**クライアントサイド防御（ウェブスキミング対策）**を緊急に強化する必要があります,。
## 結論
2025年12月6日付けの脅威概況は、グローバルな最高深刻度脆弱性が公開と同時に国家レベルのAPTグループによって悪用される「防御の時間的猶予の喪失」が常態化しつつあることを証明しています,。この状況は、パッチ管理の自動化と迅速な脅威インテリジェンス統合の必要性を強調します。同時に、国内のインシデント事例（駿河屋）は、決済システムにおけるセキュリティコード漏洩というコンプライアンス上の重大な失敗が、依然として大規模な顧客リスクをもたらしていることを示しており、基本的なセキュリティ衛生の徹底とウェブサプライチェーンの防御強化が不可欠です,。
