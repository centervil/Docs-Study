# 2026年1月28日 サイバーセキュリティニュースまとめ

## 主要ポイント
* **大手ブランドを標的とした大規模な産業スパイ活動:** Nike（1.4TB）やTarget（860GB）から機密情報やソースコードが窃取され、知的財産の流出による長期的な競争力低下が懸念されています。
* **認証基盤とアイデンティティを狙う高度な攻撃:** FortinetのSSOバイパス脆弱性（CVE-2026-24858）の悪用や、ShinyHuntersによるVishing（ボイスフィッシング）を用いたMFA突破が深刻化しています。
* **設定ミスとAIの武器化による脅威の加速:** 70万人の市民データが露出したIDHSの事例や、AIによってわずか1週間で開発されたマルウェア「VoidLink」など、防御側の管理不備と攻撃側の自動化が顕著です。

## インフラと脆弱性
重要インフラ製品や広く普及したソフトウェアに深刻な脆弱性が発見されています。**Fortinet**は、FortiOSのSSO（シングルサインオン）に関連する致命的な認証バイパス脆弱性（CVE-2026-24858）を修正しましたが、すでに野放しでの悪用が確認されています。また、**Cisco Unified Communications Manager**でもRCE（リモートコード実行）脆弱性が悪用されており、CISAが警告を発しています。オープンソースでは、Grist-Coreの「Cellbreak」（CVE-2026-24002）や、Node.jsライブラリ（vm2, binary-parser）の脆弱性が、サンドボックス回避やコード実行を可能にするとして注意喚起されています。

## 産業スパイとデータ侵害
企業の核心的な知的財産を直接狙う攻撃が急増しています。**Nike**は、恐喝グループ「WorldLeaks」によって、研究開発（R&D）や製造ワークフローを含む1.4TBのデータが窃取・公開された疑いで調査を進めています。**Target**では、従業員端末へのインフォスティーラー感染を起点に、約3〜4ヶ月にわたって860GBのソースコードや開発文書が密かに持ち出されていました。国内では、**ソフトバンク**のシステム不具合による個人情報誤表示や、**スマレジ**の外部アプリ経由でのデータ流出など、不十分な設定管理に起因する事故が報告されています。

## AIとアイデンティティの脅威
攻撃手法はアイデンティティ中心（Identity-centric）へと移行しており、AIがその触媒となっています。**ShinyHunters**は、ITサポートを装うVishingを通じてGoogleやOktaの認証情報を窃取し、MFAをバイパスしてSaaS環境へ侵入しています。開発環境も標的となっており、**VS Code Marketplace**ではAIアシスタントを装いながらマルウェアを配布する拡張機能が発見されました。また、北朝鮮の**Konni**がAIを用いて開発者向けのマルウェアを構築するなど、LLMが攻撃の工業化を支える実態が浮き彫りになっています。

## ニュース紹介
### 1. Targetのソースコード860GBが窃取、数ヶ月間気づかれず
米小売大手Targetから約860GBのソースコードと内部開発者向けドキュメントがネット上に流出しました。調査によると、2025年9月末に従業員のワークステーションがインフォスティーラーに感染したことが発端とみられます。攻撃者は盗んだ資格情報を用いてIAM、Jira、Confluenceなどの内部サービスへ自由にアクセスし、地理的な異常や大量のクローニング活動が行われていたにもかかわらず、約3〜4ヶ月間検知されませんでした。流出したデータには、CI/CDパイプラインや技術スタックのメタデータが含まれており、今後の標的型攻撃の「ロードマップ」として悪用されるリスクがあります。
[出典: SC Media - 860GB of Target source code stolen. No one knows who did it]

### 2. Nikeを狙うWorldLeaks、1.4TBの機密データを公開
ランサムウェアグループから転換した「WorldLeaks」が、Nikeから1.4TB（約18万8,000ファイル）の内部データを窃取したと主張し、サンプルをダークウェブに公開しました。公開されたデータには、製品デザイン、材料表（BoM）、製造評価、サプライチェーンのワークフローなどが含まれています。顧客の個人データは含まれていないとされていますが、知的財産の流出は偽造品の製造や競合他社による模倣、今後の製品リリース計画の妨害につながる致命的な脅威です。
[出典: HSToday - Data Thieves Claim They Ran Off With 1.4TB of Data From Nike / FireCompass - Nike Data Breach by World Leaks Ransomware Gang]

### 3. FortiOSのSSOバイパスゼロデイ脆弱性が悪用中
Fortinetは、FortiOS、FortiManager、FortiAnalyzerに影響する緊急の認証バイパス脆弱性（CVE-2026-24858）を修正しました。この脆弱性はSSO（シングルサインオン）機能の実装不備に起因し、FortiCloudアカウントを持つ攻撃者が、他人の登録済みデバイスに不正ログインすることを可能にします。すでに自動化された攻撃クラスターが確認されており、ファイアウォールの設定ファイルが数秒のうちにエクスポートされる被害も報告されています。
[出典: The Hacker News - Fortinet Patches CVE-2026-24858 After Active FortiOS SSO Exploitation Detected / SecurityWeek - Fortinet patches actively exploited FortiOS SSO auth bypass]

## 深掘り
### アイデンティティと開発環境：新たな攻撃の主戦場
2026年初頭の攻撃トレンドは、従来の脆弱性悪用から「アイデンティティの悪用」と「開発エコシステムの侵害」へと明確にシフトしています。

#### アイデンティティは「新しい制御プレーン」
ShinyHuntersやTargetの事例が示すように、攻撃者はCVE（脆弱性番号）を突くよりも、特権を持つユーザーの「アイデンティティ」を乗っ取ることに注力しています。特にVishing（ボイスフィッシング）とリアルタイムのフィッシングキットを組み合わせることで、従来の多要素認証（MFA）を無効化し、SaaS環境全体へのアクセス権を確保する「アイデンティティ戦争」が勃発しています。

#### AIによる攻撃サイクルの短縮
マルウェア「VoidLink」は、AIモデルがコードだけでなく開発計画やテスト手順までも生成する「仕様駆動開発（SDD）」によって構築されました。これにより、少人数の攻撃者でも高度で洗練された攻撃インフラを迅速に量産することが可能になっています。また、VS Code拡張機能やPyPIパッケージの「毒入れ（ポイズニング）」による開発者ワークステーションの侵害は、組織のコアシステムへの最も効率的なショートカットとして定着しつつあります。

### [関連知識]
#### [用語解説]
*   **Vishing (ボイスフィッシング):** 電話や音声クローンを用いて相手を欺き、認証情報を盗み出す手口。アイデンティティ侵害の主要な手段となっています。
*   **Infostealer (インフォスティーラー):** 端末からブラウザのCookie、パスワード、セッションデータを窃取することに特化したマルウェア。Target等の大規模侵害の起点となりました。
*   **CTEM (Continuous Threat Exposure Management):** 脅威と脆弱性を個別に管理するのではなく、実際の環境における「攻撃パス」を継続的に評価・修正する管理手法。

#### [今後の展望]
2026年は「AIエージェント」による事故や侵害が、人的ミスを超えてデータ漏洩の主要原因になると予測されています。企業はパッチ管理に加え、アイデンティティ脅威検知・応答（ITDR）を導入し、特権アカウントや開発環境の振る舞いをリアルタイムで監視する「ゼロトラスト」の徹底が不可欠です。

## 結論
最新のインシデント群は、企業のセキュリティ境界が、物理的なネットワークから「アイデンティティ」と「サプライチェーン（開発環境）」へと完全に移行したことを裏付けています。NikeやTargetの事例は、顧客データ保護だけでなく「ビジネスの核心である知的財産」を守ることが、もはや企業の存続に関わる経営課題であることを示しています。
