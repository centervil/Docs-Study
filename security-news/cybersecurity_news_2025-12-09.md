# 2025年12月9日 サイバーセキュリティニュースまとめ
## 主要ポイント
*   **React Server Components (RSC) の重大なRCE脆弱性（CVE-2025-55182）** が活発に悪用されており、CVSSスコアは最高の10.0である,。パッチ適用と侵害痕跡の調査が最優先事項となっている,。
*   中国政府支援が疑われる攻撃グループが、VMware vSphere環境のvCenterやESXiを標的とする高度なバックドア**「BRICKSTORM」**を使用し、長期的なサイバースパイ活動を展開している,,。
*   国内では、デジタル貿易赤字の是正と経済安全保障の強化を目指し、**日本サイバーセキュリティ産業振興コミュニティ（NCPC）**が設立され、国内セキュリティ産業の自立と競争力向上を牽引することが期待されている,,。

## 重大脆弱性
React Server Components (RSC) のコア機能である「Flightプロトコル」の実装に起因する、**認証不要のリモートコード実行（RCE）の欠陥（CVE-2025-55182）**が確認された,。この脆弱性は、細工されたHTTPリクエストのみで実行可能であり、CVSSスコア10.0と評価されている,。Next.js 15.x/16.x（App Router）を含むReact 19エコシステム全体に影響が及び、Next.jsの標準構成であっても開発者のコード変更なしに悪用される可能性がある,,。既にPoCが公開され、Wiz ResearchやUnit 42などのセキュリティベンダーによって、実環境でのクラウド資格情報の窃取や暗号資産マイニング、バックドアの展開を伴う悪用事例が確認されている,,,。影響を受けるReactパッケージ（`react-server-dom-webpack`、`-parcel`、`-turbopack`）はバージョン19.0.0、19.1.0-19.1.1、19.2.0などであり、**即座に修正済みバージョン（19.0.1、19.1.2、19.2.1など）へアップグレードすることが唯一の決定的な緩和策**とされている,,。

## 国際脅威分析
米国CISA、NSA、カナダ・サイバーセキュリティセンターが、中国政府支援が疑われる攻撃グループが利用する新たなバックドア**「BRICKSTORM」**について公表した,。このマルウェアはGo言語で実装されており、特に**VMware vSphere環境（vCenterサーバーやESXi）を標的**とし、仮想化基盤の制御プレーンに長期潜伏型のサイバースパイ活動を行うことを目的としている,。BRICKSTORMは、C2通信に**DNS-over-HTTPS (DoH)**を利用し、WebSocketとTLSによる多層暗号化を用いることで、従来の防御メカニズムを回避する,,。さらに、仮想環境に特化した機能として、**VSOCK（仮想ソケットプロトコル）**を悪用し、物理ネットワークを介さずにVM間通信を行うことが可能である,。攻撃チェーンは、DMZ上のWebサーバー侵害からサービスアカウント認証情報の窃取を経て、vCenterへ横展開し、BRICKSTORMを配置するという段階的な戦略が確認されている,。

## 国内戦略政策
2025年12月9日、日本におけるサイバーセキュリティの発展と産業振興を目的とした**「日本サイバーセキュリティ産業振興コミュニティ」（NCPC）**が設立された。設立の背景には、サイバー攻撃の深刻化、経済安全保障推進法の成立、そしてデジタル貿易赤字と国外製品への高い依存という構造的な課題がある,。NCPCは、**「日本の守る力を興す力へ」**をビジョンに掲げ、国内セキュリティ製品・サービスを体系的に**マッピングしスコアリングする枠組みの構築**、セキュア設定ガイドの共有、国内外への展開支援、政策提言などを主要な活動として行う予定である,,。この取り組みは、国内産業の競争力向上と、地政学的リスクが高まる中での日本の**デジタル主権の確保**に貢献することが期待される。

# 2025年12月9日 サイバーセキュリティニュース詳細
## ニュース紹介
### 1. React Server Componentsの重大な認証不要RCE脆弱性（CVE-2025-55182）
React Server Components (RSC) のFlightプロトコルにおける信頼できないデータのデシリアライゼーション（CWE-502）の脆弱性（CVE-2025-55182）が、CVSS 10.0の最高評価を受けている,。この欠陥は、サーバーが細工された不正なHTTPペイロードを受信した際にデータの構造検証に失敗し、攻撃者制御下のデータがサーバー側の実行ロジックに影響を与え、任意の特権JavaScriptコードの実行を可能にする,,。
この脆弱性は、React 19.0、19.1、19.2の各バージョンや、Next.js 15.x、16.x（App Router利用時）に影響を与え、Next.jsの標準構成であっても開発者によるコード変更なしに悪用が可能であり、その信頼性はテストでほぼ100%であることが示されている,,,。
既にUnit 42は、この脆弱性を悪用した後のポストエクスプロイト活動を観測しており、偵察活動、Cobalt Strikeのインストール試行、暗号資産マイニングソフトウェアの展開、EtherRATやBPFDoorなどの高度なマルウェアの実行が確認されている,,,,,。また、中国関連の脅威アクタークラスターCL-STA-1015によるSNOWLIGHTおよびVShellトロイの木馬のインストールも観測されている,,。
出典: Unit 42 - Exploitation of Critical Vulnerability in React Server Components (Updated December 8),,,,,,,,,,,,,

### 2. 中国政府支援グループによる仮想化基盤を狙うBRICKSTORMバックドア攻撃キャンペーン
中国（PRC）政府支援が疑われる攻撃グループが、VMware vSphere環境（vCenter/ESXi）を主要な標的とするGo言語で実装された新たなバックドア**「BRICKSTORM」**を利用したサイバースパイ活動の詳細が公表された,,。このキャンペーンでは、DMZ上のWebサーバー侵害を起点とし、サービスアカウントの認証情報を悪用することで、ドメインコントローラやADFSサーバーを経て、最終的にvCenterサーバーへアクセスを拡大させていた,。
BRICKSTORMは、仮想化基盤上で長期的なアクセスを維持するために設計されており、検出を回避する高度な通信技術を持つ,。特に、C2通信には**DNS-over-HTTPS (DoH)**が利用され、従来の境界防御やネットワークインスペクションを回避する,。さらに、仮想環境に特化した機能として、**VSOCKプロトコル**を悪用し、物理ネットワークを介さずにVM間またはホストとの間で密かに通信を行うことができる,。攻撃者は侵害したvSphere環境で、仮想マシンのスナップショット窃取や隠しVMの作成といった仮想化レイヤー固有の機能を利用していたことが観測されている,。
出典: サイバーセキュリティ脅威・政策動向分析レポート (2025年12月9日付),,,

### 3. 日本サイバーセキュリティ産業振興コミュニティ（NCPC）設立
2025年12月9日、日本国内のセキュリティ製品・サービス提供企業が連携し、セキュリティ強化と経済安全保障を充実させることを目的とした**日本サイバーセキュリティ産業振興コミュニティ（NCPC）**が設立された,。これは、デジタル貿易赤字の拡大と国外製品への高い依存という課題に対応し、国内セキュリティ産業の自立と競争力向上を喫緊の課題として捉えた、国家戦略（経済産業省の「サイバーセキュリティ産業振興戦略」）に基づく取り組みである,。
NCPCの活動には、日本のセキュリティ製品・サービスを体系的に**マッピングしスコアリングする枠組みの構築**、セキュア設定ガイドの共有、国内外への製品展開支援、そして政策提言が含まれる,。このコミュニティは、今後の公的機関や重要インフラ部門のセキュリティ製品調達戦略に影響を与え、**デジタル主権の確保**に貢献することが期待されている。

## 深掘り
### CVE-2025-55182の悪用活動とクラウドへの影響
React Server ComponentsのRCE脆弱性（CVE-2025-55182）が悪用された後、攻撃者はアプリケーションサーバーの侵害に留まらず、その基盤となるクラウドインフラストラクチャ全体を標的としている。この脆弱性は、認証プロセスが不要（unauthenticated）で、ほぼ100%の信頼性で任意のコード実行を可能にするため、攻撃者にとって「信頼できるシステム全体のバイパス」を提供する,。

#### 観測されたポストエクスプロイト活動
Unit 42やWiz Researchは、RCE成功後の攻撃者のTTPsが極めて洗練されていることを確認している。
*   **迅速な偵察:** Base64エンコードされたコマンドを利用し、侵害直後にOSやアーキテクチャの指紋採取（`uname`）、特権レベルの検証（`id`）、ネットワークインターフェースのマッピング（`hostname`）を迅速に実施している,,。
*   **クレデンシャル窃取:** 攻撃者はファイルシステム内でクラウド資格情報の設定ファイルや機密性の高い認証情報ファイルを列挙し、窃取を試みる,,。これは、脆弱性がクラウド環境のNext.jsアプリを標的とした場合、アプリケーション層の侵害が直ちにIAMの制御権を脅かし、クラウドインフラ全体へ水平展開する経路を提供することを意味する。
*   **国家支援が疑われる活動との関連:** 北朝鮮（DPRK）のContagious Interviewツールと重複する活動（EtherRATの使用など）や、中国関連の脅威アクターCL-STA-1015によるSNOWLIGHT/VShellトロイの木馬のインストールも確認されている,,。

### 関連知識
#### 用語解説
*   **React Server Components (RSC):** Reactが提供するアーキテクチャの一つで、コンポーネントの一部をサーバー側で実行し、結果をクライアントにストリーミングすることで、パフォーマンス向上と効率的なデータ取得を狙う仕組み,。
*   **デシリアライゼーション（CWE-502）:** ネットワーク経由などで受け取ったシリアライズされたデータを、プログラム内部のオブジェクトや構造に復元する処理。CWE-502は、信頼できない入力を安全でない方法でデシリアライズすることによる脆弱性カテゴリである,。
*   **リモートコード実行（RCE）：** 攻撃者がネットワーク経由で標的のサーバー上で任意のコードを実行できてしまう状態,。
*   **Flightプロトコル:** RSCがクライアントとサーバー間でデータをやり取りするために使用する、独自のデシリアライゼーション処理を含むプロトコル,,。
*   **PoC（Proof of Concept）：** 脆弱性が実際に悪用可能であることを証明するための概念実証コード。

#### 今後の展望
この脆弱性の性質は「決定論的なロジック上の欠陥」であり、広範なNext.js/Reactの普及率と相まって、**機密性の高い内部データへの直接的な導管**を作り出した。組織は、パッチ適用に加えて、アプリケーションが使用する実行環境（コンテナやVM）のIAMロール権限を最小化し、不審なクラウドAPIコールを監視するDevSecOps体制の強化が必須となる,。また、今後、モダンなフレームワークを選定する際には、パフォーマンスや開発者体験だけでなく、デフォルト設定でのセキュリティやプロトコル設計の安全性が重要な評価軸となると考えられる。

## 結論
CVE-2025-55182の即時かつ広範囲にわたる悪用は、脅威アクターが機会を捉える速度がいかに速いかを浮き彫りにした。この重大なRCE脆弱性への緊急対応（即時パッチ適用と侵害痕跡の徹底調査）が、組織のセキュリティ部門にとって現在の最優先事項である,。特にクラウド環境でNext.jsを利用している場合、アプリケーションの侵害がクラウド資格情報の窃取に直結するリスクがあるため、パッチ適用に加えて、実行環境の権限最小化と監視強化が不可欠である,。同時に、BRICKSTORM攻撃キャンペーンが示すように、国家主導の脅威は仮想化基盤という重要インフラを狙う高度な手法を採用しており、従来のネットワーク監視を回避するDoHやVSOCKプロトコルの監視導入を含む戦略的な防御体制への移行が求められる,,。
